# PROJECT_CONTEXT.md  
Praxis Telichko – Website

## 1. Projektübersicht

Dieses Projekt ist die offizielle Website der ärztlichen Praxis **Telichko** mit zwei Praxisstandorten.  
Die Website dient primär der Information von Patienten und der vertrauensvollen Darstellung der Praxis.

Ziele der Website:
- Bereitstellung klarer Informationen zu Leistungen, Standorten und Öffnungszeiten
- Unterstützung der Terminvereinbarung
- Seriosität, Vertrauen und medizinische Professionalität vermitteln
- Gute Nutzbarkeit für ältere Patienten
- Mehrsprachige Nutzung (z. B. Deutsch / Russisch)

## 2. Zielgruppe

- Bestandspatienten
- Neupatienten
- Ältere Patienten
- Russischsprachige Patienten

## 3. Technischer Stack (Ist-Zustand)

- Hosting: GitHub Pages
- Technologie: Statisches HTML / CSS / JavaScript
- CSS Framework: Bootstrap (Template-basiert)
- Keine serverseitige Logik
- Formulare clientseitig (ggf. Weiterleitung oder JS-basierter Versand)

## 4. Projektstruktur / Seiten

Aktuelle Seiten bzw. Sektionen:
- Startseite
- Praxisvorstellung
- Leistungen
- Team / Ärztin
- Standorte (zwei Praxen)
- Öffnungszeiten
- Kontakt / Terminvereinbarung

## 5. Inhaltliche Leitlinien (medizinischer Kontext)

- Keine Heilversprechen
- Keine konkreten Diagnosen oder Therapieempfehlungen
- Sachliche, neutrale und vertrauenswürdige Formulierungen
- Patientenorientierte Sprache
- Medizinische Inhalte nur auf allgemeiner Informationsebene

## 6. Tonalität & Sprache

- Ruhig
- Seriös
- Klar und verständlich
- Keine werblichen oder reißerischen Aussagen
- Gut lesbar für ältere Nutzer

## 7. Recht & Datenschutz

- DSGVO beachten
- Keine Tracking- oder Analyse-Tools ohne Einwilligung
- Formulare nur mit notwendigem Minimum an Daten
- Datenschutzerklärung muss korrekt verlinkt sein

## 8. UI/UX-Richtlinien

- Klare Navigation
- Große, gut lesbare Schrift
- Hohe Kontraste
- Konsistentes Layout auf allen Seiten
- Mobile Nutzung berücksichtigen
- Barrierearme Gestaltung (Labels, klare Buttons, verständliche Texte)

## 9. Code- und Stilkonventionen

- Sauberes, semantisches HTML
- Bootstrap-Komponenten bevorzugen
- Keine unnötigen externen Bibliotheken
- Kein Inline-CSS, sofern vermeidbar
- JavaScript klar strukturiert und minimal halten

## 10. SEO-Grundlagen

- Sinnvolle Title-Tags pro Seite
- Meta-Descriptions setzen
- Alt-Texte für Bilder
- Klare Überschriftenhierarchie (H1–H3)
- Schnelle Ladezeiten
